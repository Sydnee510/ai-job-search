# Writing Voice Rules

> **Applies to every user-facing output generated by any `ai-job-*` skill** — resumes, cover letters, interview briefs, pipeline summaries, recruiter emails. Does NOT apply to internal chat responses between Claude and the user (that's conversation, not artifact).
>
> Each skill's `SKILL.md` references this file in its "Critical behaviors" section. When generating output artifacts, re-read these rules before writing.

## The #1 rule: NO EM-DASHES IN PROSE

**Do not use the em-dash character (`—`, U+2014) in generated documents.** It's the most reliable AI-generated text tell in 2026. Recruiters and hiring managers scan for it.

**Substitutions (use whichever fits best):**

| Em-dash usage | Replace with |
|---|---|
| Parenthetical aside: "I shipped X — a voice system — for Y customers" | Comma pair: "I shipped X, a voice system, for Y customers" OR restructure into two sentences |
| Emphasis: "I did one thing — shipped the product" | Colon: "I did one thing: shipped the product" OR period |
| Interrupted thought: "We hit the deadline — barely" | Period: "We hit the deadline. Barely." |
| List intro: "The plan — tests, build, ship" | Colon: "The plan: tests, build, ship" |

**En-dashes (`–`, U+2013) are allowed in date ranges only** (e.g. `04/2025 – present`, `2020–2023`). Not in prose sentences.

**Regular hyphens (`-`, U+002D) are fine everywhere** (compound words, URLs, etc.).

## The #2 rule: banned AI-tell phrases

These phrases get cut, even if they're technically accurate. They read as AI-generated to 2026 recruiters:

- "I'm excited to apply"
- "I believe my experience"
- "I'm a perfect fit"
- "passionate about"
- "cutting-edge"
- "state-of-the-art"
- "seamlessly"
- "leverage" (as a verb — "use" is fine)
- "synergy"
- "best-in-class"
- "in today's fast-paced world"
- "proven track record of delivering"
- "ecosystem" (when you mean "product" or "platform")
- "robust"
- "scalable solutions"
- "holistic"
- "at scale" as a throwaway modifier (OK only if you have a specific scale number)
- "transformative"
- "revolutionize"

## The #3 rule: formatting AI-tells

These are structural patterns LLMs default to. Break them.

- **Don't bold random phrases in prose.** Bold is only for section labels ("**AI Systems:** ...") and names. No mid-sentence emphasis.
- **Don't use bullet lists for narrative.** Resume bullets are fine. Cover letter bullets are a tell — write in paragraphs.
- **Don't start a cover letter with "Dear Hiring Manager," unless you know no name.** If you can find the HM or recruiter's name, use it.
- **Don't hedge.** "I might be a fit" → "I'm a fit because X." Own the claim.
- **Don't nest more than 2 clauses in a sentence.** Long compound sentences with multiple ideas connected by commas are LLM-default. Break into shorter sentences.

## The #4 rule: authorial-voice guidance

Preserve this voice across all outputs:

- **Dense, metric-heavy.** Every claim has a number when one exists. If no number, don't pad with adjectives.
- **Active voice, past-tense verbs: led, shipped, architected, owned, drove, built, scaled, wrote, launched.**
- **Specific > general.** "Anthropic Claude via streaming API" beats "modern AI technology". "500+ customers" beats "many customers".
- **First-person, singular.** "I did X" in cover letters; no bullet starts with a verb in resume (bullets are implicitly "I").
- **No throat-clearing.** Cut any opening sentence that doesn't move the argument forward. The first sentence of a cover letter is the hook, not a greeting.

## Self-check before finalizing any output

Run this check:

1. Ctrl-F for `—` (em-dash). **If any exist, rewrite them.** Every single one.
2. Ctrl-F for each banned phrase. **If any exist, rewrite or delete.**
3. Count consecutive sentences that start with "I". Aim for ≤3 in a row.
4. Read aloud. If a sentence sounds like a LinkedIn post, rewrite it.
5. Word count: cover letters cap at 300. Resume bullets cap at 2 lines each.

If a check fails, fix before saving.
