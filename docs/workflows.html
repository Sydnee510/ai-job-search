ai-job-search — Skill Workflows

Five skills, one per job-to-be-done. Click any row to see its workflow diagram.

ai-job-discover
Find open roles matching your target_job_titles. Polls Greenhouse / Lever / Ashby + runs targeted WebSearch. Creates tracker rows for new roles.
View workflow →
ai-job-apply
Turn a JD into a tailored resume + cover letter PDF package. Analyzes fit, picks stage-adaptive bullet variants, drafts cover letter from story bank, logs to tracker.
View workflow →
ai-job-interview-prep
Generate a one-page interview brief. 5 likely questions with story mappings, 3 smart questions to ask, a 30/60/90 sketch, and gap flags.
View workflow →
ai-job-pipeline
Review the state of all applications in one place. Totals by stage, stale flags, weekly review mode, recommended next actions.
View workflow →
ai-job-submit (STUB)
Deliberate stub. Humans click Submit, always. This skill surfaces the manual-submission checklist and refuses to auto-send anything.
View workflow →

Typical end-to-end flow

Weekly rhythm:
1. ai-job-discover
→
2. ai-job-pipeline (review)
→
3. ai-job-apply (for top 2–5)
→
4. (human clicks Submit)
→
5. ai-job-pipeline (log applied)
→
6. ai-job-interview-prep (when scheduled)

This works for any role

The project defaults lean AI because that's where the original author targeted. Everything — ATS polling, resume tailoring, cover-letter drafting, interview-prep, pipeline review — is role-agnostic. Use it for AI Engineer, PM, UX Designer, Software Engineer, TPM, Cybersecurity Analyst, Data Scientist, Developer Advocate, Researcher, or anything else. Edit knowledge/user_profile.json to set your target titles. See knowledge/user_profile_schema.md for a starter title list per role family.
